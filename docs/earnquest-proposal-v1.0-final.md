## Executive Summary

EarnQuest는 **아이들에게 동기부여를 주는 가족 리워드 플랫폼**입니다.

- **핵심 루프**: 태스크 완료 → 포인트 적립 → 스토어에서 리워드 구매
- **타겟 사용자**: 초등학생 자녀를 둔 부모 (핵심: 8-11세)
- **핵심 차별점**: 스크린타임 통합 리워드 + 교육 앱 연동 + 심리학 기반 설계

## 1. Core Flow

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   TASKS              POINTS              STORE              │
│   태스크      →      포인트 적립    →    스토어 구매         │
│                                                             │
│   • 디폴트 제공      • 완료 요청        • 스크린 타임        │
│   • 부모 커스텀      • 부모 확인        • 경험/자율권        │
│   • 아이 제안        • 또는 자동        • 저축               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 핵심 가치
> **아이들에게 동기부여를 준다**

이것이 EarnQuest의 존재 이유입니다. 나머지 모든 설계는 이 핵심을 지키기 위한 것입니다.

## 2. Task System

### 2.1 태스크 카테고리

| 카테고리 | 설명 | 디폴트 포함 | 포인트 특성 |
|---------|------|------------|------------|
| **학습** | 숙제, 독서, 학습 앱 | ✅ YES | 정규 포인트 |
| **생활** | 방 정리, 식사 정리 | ✅ YES | 정규 포인트 |
| **건강** | 운동, 자기관리 | ✅ YES | 정규 포인트 |
| **친사회적** | 가족 돕기, 배려 | ⚠️ 특별 | 인정 시스템 (§5 참조) |

### 2.2 디폴트 태스크 (8-11세 기준)

| 카테고리 | 태스크 | 포인트 | 승인 방식 | 비고 |
|---------|--------|--------|----------|------|
| 학습 | 숙제 완료 | 50 | 부모 확인 | 체크리스트 옵션 |
| 학습 | 독서 20분 | 30 | 타이머 | 자동 승인 가능 |
| 생활 | 방 정리 | 30 | 부모 확인 | 사진 옵션 |
| 생활 | 식사 후 정리 | 20 | 자동 | 신뢰 태스크 |
| 건강 | 운동 30분 | 40 | 부모 확인 | - |

### 2.3 태스크 속성

```
Task {
  id: string
  name: string
  category: "학습" | "생활" | "건강"
  points: number
  approvalType: "parent" | "auto" | "timer" | "checklist" | "app"
  checklist?: string[]        // 체크리스트 항목
  timerMinutes?: number       // 타이머 시간
  photoRequired?: boolean     // 사진 필요 여부
  trustLevel?: number         // 신뢰 레벨 (§7 참조)
  ageGroup: "5-7" | "8-11" | "12-14"
}
```

### 2.4 아이 선택권 (20-40%)

- **태스크 제안**: 아이가 새 태스크 제안 → 부모 승인 후 추가
- **포인트 협상**: 아이가 포인트 조정 요청 → 부모 승인/거절/역제안

## 3. Point System

### 3.1 포인트 밸런스 (8-11세 기준)

| 기간 | 목표 포인트 | 설명 |
|-----|------------|------|
| 하루 | 100-120 pts | 태스크 3-4개 완료 시 |
| 주간 | 500-600 pts | 평일 5일 열심히 하면 |

### 3.2 설계 원칙

```
"하루 열심히 하면 → 당일 작은 보상 1개 (스크린 30분)"
"일주일 모으면 → 괜찮은 보상 1개 (경험/자율권)"
"한 달 모으면 → 큰 보상 1개 (원하는 물건)"
```

### 3.3 포인트 적립 흐름

```
아이: 태스크 완료 → "완료" 버튼
     ↓
시스템: 승인 방식에 따라 처리
     ↓
  ┌─ 자동 승인 → 즉시 포인트 적립
  ├─ 타이머 → 시간 충족 시 적립
  ├─ 체크리스트 → 모두 체크 시 적립
  └─ 부모 확인 → 부모 승인 후 적립 (또는 24h 후 자동)
```

## 4. Store System

### 4.1 디폴트 스토어 (8-11세 기준)

**배치 순서** (비스크린 우선):

| 순서 | 카테고리 | 리워드 | 포인트 | 설명 |
|-----|---------|--------|--------|------|
| 1 | **자율권** | 저녁 메뉴 결정권 | 80 | "오늘 뭐 먹을지 네가 정해!" |
| 2 | **자율권** | 취침 30분 연장 | 120 | 주말 한정 |
| 3 | **경험** | 부모와 보드게임 30분 | 60 | "엄마/아빠랑 둘이서만" |
| 4 | **경험** | 아이스크림 | 150 | - |
| 5 | **스크린** | 게임 30분 | 100 | 주간 예산 적용 |
| 6 | **스크린** | 유튜브 30분 | 100 | 주간 예산 적용 |
| 7 | **저축** | $1 저축 | 150 | 저축통에 적립 |

### 4.2 스크린 리워드 관리

```
주간 스크린 예산: 최대 5시간 (디폴트, 부모 조절 가능)

예산 소진 시:
"이번 주 스크린 예산을 다 썼어요! 다른 리워드는 어때요?"
→ 비스크린 리워드 추천
```

### 4.3 리워드 속성

```
Reward {
  id: string
  name: string
  category: "자율권" | "경험" | "스크린" | "저축" | "물건"
  points: number
  description: string           // 감정적 언어 포함
  weeklyLimit?: number          // 주간 구매 제한
  requiresParentAction: boolean // 부모 실행 필요 여부
}
```

### 4.4 리워드 구매 흐름

```
아이: 리워드 선택 → "구매" 버튼
     ↓
시스템: 포인트 충분? 주간 예산 내?
     ↓
  YES → "티켓 발행" (즉시 의미 부여)
        부모에게 알림: "안나가 [아이스크림] 티켓을 구매했어요"
     ↓
부모: 실제 리워드 지급 (나중에 OK)
     ↓
시스템: "지급 완료" 체크
```

## 5. Kindness System (친사회적 행동)

### 5.1 설계 철학

> **"친사회적 행동에 정규 포인트를 주면 '착한 일'이 '알바'가 된다"**

심리학 연구에 따르면, 도움/나눔 같은 친사회적 행동에 물질적 보상을 붙이면 내적 동기가 감소합니다. EarnQuest는 이를 **인정 시스템**으로 분리합니다.

### 5.2 인정 시스템 구조

```
┌─────────────────────────────────────────────────────────────┐
│              친사회적 행동 = "인정 시스템"                    │
├─────────────────────────────────────────────────────────────┤
│ 1. 감사 카드 (기본)                                          │
│    • 가족 누구나 "고마워" 카드 보낼 수 있음                   │
│    • 포인트 0, 순수한 감사 표현                              │
│    • 카드 내용: "오늘 동생 숙제 도와줘서 고마워!"            │
│                                                             │
│ 2. 친절 배지 (시각화)                                        │
│    • 감사 카드 5개 = 🏅 친절 배지 1개                        │
│    • 배지 컬렉션으로 모으는 재미                             │
│    • 포인트 가치 없음, 순수한 인정                           │
│                                                             │
│ 3. 주간 친절 보너스 (희소 보상)                              │
│    • 매주 일요일 "가족 정산" 시간에                          │
│    • 부모가 "이번 주 가장 감동받은 친절한 행동" 선택         │
│    • 선택된 행동에 서프라이즈 30-50 pts                      │
│    • 예상 못한 보상 = 내적 동기 보존                         │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 부모 opt-in 옵션

부모가 원하면 친사회적 행동을 정규 태스크로 추가할 수 있습니다.
단, 시스템이 경고 메시지를 표시합니다:

```
⚠️ 전문가 안내
"친사회적 행동(동생 돕기, 배려 등)에 정규 포인트를 주면
아이가 '보상을 위해' 하게 될 수 있습니다.
대신 '인정 시스템'으로 감사와 칭찬을 전하는 것을 권장합니다.
그래도 포인트 태스크로 추가하시겠습니까?"

[인정 시스템 사용 (권장)]  [포인트 태스크로 추가]
```

## 6. Approval System

### 6.1 설계 철학

> **"거절"이라는 단어는 사용하지 않는다**

"거절"은 아이에게 처벌로 느껴집니다. 대신 **"다시 확인"**으로 학습 기회를 제공합니다.

### 6.2 상태 모델 (3단계)

| 상태 | 아이콘 | 아이에게 보이는 문구 | 설명 |
|-----|-------|---------------------|------|
| **확인 완료** | ✅ | "잘했어! 포인트가 들어왔어요" | 승인됨 |
| **다시 확인** | 🔁 | "조금만 더 하면 포인트 받을 수 있어!" | Fix 요청 |
| **확인 대기** | ⏳ | "부모님이 확인 중이에요" | 대기 중 |

### 6.3 "다시 확인" (Fix 요청) UX

**부모 화면:**
```
[안나] 방 정리 완료 요청

[✅ 확인 완료]  [🔁 다시 확인]  [⏳ 나중에]

"다시 확인" 선택 시 → 원클릭 템플릿:
┌─────────────────────────────────────┐
│ 조금만 더 해볼까? (최대 2개 선택)   │
│                                     │
│ ☐ 바닥에 있는 옷 정리해줘 👕        │
│ ☐ 책상 위만 한 번 더 정리! 📚      │
│ ☐ 침대 정리가 빠졌어 🛏️            │
│ ☐ 직접 입력...                     │
│                                     │
│ [보내기]                            │
└─────────────────────────────────────┘
```

**아이 화면:**
```
┌─────────────────────────────────────┐
│ 🔁 조금만 더!                       │
│                                     │
│ 부모님이 응원 메시지를 보냈어요:    │
│                                     │
│ "바닥에 있는 옷 정리해줘 👕"        │
│ "책상 위만 한 번 더 정리! 📚"      │
│                                     │
│ 다 하면 바로 포인트 받을 수 있어!   │
│                                     │
│ [다시 완료 요청]                    │
└─────────────────────────────────────┘
```

### 6.4 자동 승인

| 조건 | 동작 |
|-----|------|
| 24시간 경과 | 자동 승인 (디폴트) |
| 신뢰 태스크 | 즉시 자동 승인 |
| 체크리스트 완료 | 자동 승인 |
| 타이머 완료 | 자동 승인 |

### 6.5 분쟁 반복 시

```
"다시 확인"이 3회 이상 반복되면:

시스템 → 부모에게 제안:
"이 태스크 기준이 아이에게 모호할 수 있어요.
기준을 더 구체적으로 정해볼까요?"

→ 태스크 편집 화면으로 이동
→ 체크리스트 추가 권장
```

## 7. Trust Level System

### 7.1 설계 철학

> **"벌금 대신 신뢰 레벨로 권한을 조절한다"**

포인트 차감(벌금)은 앱을 처벌 시스템으로 만듭니다. 대신 **논리적 결과**로 접근합니다:
- 정직하면 → 더 많은 자동 승인 권한
- 치팅하면 → 권한 회수 (다시 확인 필요)

### 7.2 신뢰 레벨 구조

```
┌─────────────────────────────────────────────────────────────┐
│                    신뢰 레벨 시스템                          │
├─────────────────────────────────────────────────────────────┤
│ Level 1: 시작 (모든 신규 사용자)                            │
│ • 모든 태스크 부모 확인 필요                                │
│ • 24h 자동 승인 적용                                        │
├─────────────────────────────────────────────────────────────┤
│ Level 2: 신뢰 획득 (2주 연속 정직)                          │
│ • 일부 태스크 자동 승인 (식사정리, 양치 등)                 │
│ • 타이머 기반 태스크 자동 처리                              │
├─────────────────────────────────────────────────────────────┤
│ Level 3: 높은 신뢰 (4주 연속 정직)                          │
│ • 대부분 태스크 자동 승인                                   │
│ • 부모는 샘플링 검증만 (5개 중 1개 확인)                    │
├─────────────────────────────────────────────────────────────┤
│ 🔻 신뢰 강등 (치팅 발각 시)                                 │
│ • 해당 태스크만 Level 1로 복귀                              │
│ • 메시지: "신뢰를 다시 쌓을 때까지 확인이 필요해"           │
│ • 1주일 후 레벨 복구 가능                                   │
└─────────────────────────────────────────────────────────────┘
```

### 7.3 치팅 대응 프로토콜

```
1회 발각: "다시 확인" + 대화 유도
          "이번엔 제대로 안 된 것 같아. 다시 해볼까?"

2회 발각: 해당 태스크 신뢰 모드 OFF
          "이 태스크는 당분간 확인이 필요해"

3회 발각: 태스크 쿨다운 (일시 중지)
          "잠깐 쉬고, 더 쉬운 버전으로 바꿔볼까?"
```

### 7.4 가벼운 증빙 UX

| 태스크 | 증빙 방법 | 부담 |
|-------|----------|------|
| 독서 | 타이머 + 한 문장 요약 (10초) | 낮음 |
| 숙제 | 사진 1장 또는 같이 채점 | 낮음 |
| 운동 | 운동 종류 선택 + 기분 이모지 | 매우 낮음 |
| 방 청소 | Before/After 사진 (선택) | 낮음 |

## 8. Sibling Fairness

### 8.1 설계 철학

> **"다르게 대하는 것 ≠ 불공정"**

연구에 따르면, 차등 대우 자체보다 **아이들이 그걸 공정하다고 느끼느냐**가 더 중요합니다.

### 8.2 공정성 전략

```
┌─────────────────────────────────────────────────────────────┐
│                    형제 공정성 설계                          │
├─────────────────────────────────────────────────────────────┤
│ 1. UI 비교 차단                                             │
│    • 아이 화면: 본인 포인트/내역만 표시                     │
│    • 형제 잔액/랭킹 노출 안함                               │
│    • 부모 화면에서만 전체 현황 확인                         │
├─────────────────────────────────────────────────────────────┤
│ 2. 개인 목표 분리                                           │
│    • 각 아이는 자기만의 주간 목표                           │
│    • 비교 대상: "형제"가 아니라 "나의 지난주"               │
│    • 목표 달성률(%)로 성취감 강조                           │
├─────────────────────────────────────────────────────────────┤
│ 3. 태스크 정의 분리                                         │

## 9. Parent Retention

### 9.1 설계 철학

> **"앱을 안 열어도 굴러가게 + 주간 한 번만 열면 되게"**

### 9.2 부모 이탈 방지 전략

```
┌─────────────────────────────────────────────────────────────┐
│                    부모 Retention 설계                       │
├─────────────────────────────────────────────────────────────┤
│ 1. 자동화 극대화                                            │
│    • 24h 자동 승인 (디폴트)                                 │
│    • 신뢰 태스크 즉시 자동 승인                             │
│    • 체크리스트/타이머 기반 자동 처리                       │
│    • 샘플링 검증 (5개 중 1개만 확인)                        │
├─────────────────────────────────────────────────────────────┤
│ 2. 주간 리듬                                                │
│    • 일요일 저녁 "가족 정산" 시간                           │
│      - 이번 주 가장 뿌듯한 것 1개 선택                      │
│      - 친절 보너스 지급                                     │
│      - 다음 주 태스크 1개만 조정                            │
│    • 5-10분이면 충분                                        │
├─────────────────────────────────────────────────────────────┤
│ 3. 티켓 시스템 (리워드)                                     │
│    • 아이가 구매 → 즉시 "티켓" 발행                         │
│    • 부모는 실제 실행을 나중에 해도 OK                      │
│    • 중요: 구매가 즉시 의미가 있다는 체감                   │
├─────────────────────────────────────────────────────────────┤
│ 4. 부모 동기부여                                            │
│    • 주간 요약 푸시:                                        │
│      "이번 주 안나가 5개 태스크 완료! 잔소리 줄었죠?"       │
│    • 부모 배지: "연속 7일 승인 완료"                        │
│    • 통계보다 감정형 메시지가 효과적                        │
├─────────────────────────────────────────────────────────────┤
│ 5. 이탈 감지 & 복귀                                         │
│    • 3일 미접속 → 간단 모드 제안                            │
│      "바쁘시죠? 태스크 3개로 줄여볼까요?"                   │
│    • 7일 미접속 → 부드러운 리마인드                         │
│      "안나가 기다리고 있어요"                               │
└─────────────────────────────────────────────────────────────┘
```

## 12. Artales Integration

### 12.1 개요

Artales는 Peter가 개발한 AI 영어 학습 앱입니다:
- 아이 그림 → AI가 영어 스토리 생성 → 미션 수행 → 보상

### 12.2 연동 구조

```
┌─────────────────────────────────────────────────────────────┐
│                    Artales ↔ EarnQuest                      │
├─────────────────────────────────────────────────────────────┤
│ Artales에서:                                                │
│ • 미션 완료 이벤트 발생                                     │
│ • API로 EarnQuest에 전송                                    │
│                                                             │
│ EarnQuest에서:                                              │
│ • 자동으로 35-40 pts 적립                                   │
│ • "연동 태스크" 카테고리에 기록                             │
│ • 부모 승인 불필요 (자동)                                   │
└─────────────────────────────────────────────────────────────┘
```

### 12.3 포인트 설정

| 항목 | 값 | 이유 |
|-----|---|------|
| Artales 미션 1회 | **35-40 pts** | 독서(30)보다 약간 높고, 숙제(50)보다 낮게 |
| 하루 상한 | 2회 (70-80 pts) | 자동화 폭주 방지 |

### 12.4 설계 원칙

```
1. EarnQuest는 연동 없이도 완전해야 함
   - Artales 미사용 가정에도 100% 기능 제공
   
2. Artales는 "추가 보너스 채널"로 포지셔닝
   - "치트키"가 아님
   - 다른 태스크와 비슷한 가치
   
3. 확장 가능한 구조
   - 나중에 다른 교육 앱도 동일 방식으로 연동
   - "EarnQuest Connect API"의 시초
```

### 12.5 부모 설정

```
연동 앱 관리:
┌─────────────────────────────────────┐
│ Artales                             │
│ 상태: 연결됨 ✅                     │
│ 포인트: 미션당 40 pts               │

│                                     │
│ [연결 해제]  [설정 변경]            │
└─────────────────────────────────────┘

## 15. Design Principles

### 15.1 핵심 원칙

```
1. 동기부여 > 통제
   - EarnQuest는 "감시 도구"가 아니라 "동기부여 도구"
   
2. 협상 > 명령
   - 부모-아이 관계를 "계약"으로, "권력"이 아니게
   
3. 신뢰 > 검증
   - 기본은 신뢰, 필요할 때만 검증
   
4. 습관 > 보상
   - 장기 목표는 "포인트 없이도 하는 습관"
   
5. 단순함 > 완벽함
   - 복잡한 기능보다 쉬운 사용
```

### 15.2 UX 언어 가이드

| ❌ 피할 것 | ✅ 사용할 것 |
|-----------|------------|
| 거절 | 다시 확인 |
| 벌금 | 신뢰 레벨 조정 |
| 승인 | 확인 완료 |
| 부모 승인 | 가족 확인 |
| 규칙 | 가족 약속 |
| 포인트 차감 | 권한 조정 |

### 15.3 가족 싸움 방지 원칙

```
1. 규칙의 가시화
   - "아빠가 안 주는 게 아니라, 우리 가족 약속이 이래"
   
2. 공정성 대화 도구
   - 형제 차이를 숨기지 말고, "왜 다른지" 설명하는 가이드
   
3. 실수도 학습
   - 거절/치팅도 "다시 시도 기회"로
   
4. 부모도 완벽할 필요 없음
   - 며칠 못 써도 쉽게 재시작
```

| 용어 | 정의 |
