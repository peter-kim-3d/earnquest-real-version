# Log Issue Skill

버그 수정 후 `qa/ISSUES.md`의 Issue Tracking 테이블에 기록을 추가합니다.

## 사용법

```
/log-issue
```

## 동작

1. 현재 대화에서 수정한 버그 정보를 수집:
   - **이슈**: 버그 증상 (간단히)
   - **원인**: 근본 원인
   - **수정**: 어떻게 해결했는지
   - **관련 파일**: 수정된 파일들

2. `qa/ISSUES.md` 파일의 테이블에 새 행 추가:
   ```
   | YYYY-MM-DD | 이슈명 | 원인 | 수정 | `파일1`, `파일2` |
   ```

3. 날짜는 오늘 날짜 사용 (YYYY-MM-DD 형식)

4. 관련 파일은 백틱으로 감싸기

## 예시

수정 전:
```markdown
| 날짜 | 이슈 | 원인 | 수정 | 관련 파일 |
|------|------|------|------|----------|
| 2026-01-22 | 기존 이슈 | ... | ... | ... |
```

수정 후:
```markdown
| 날짜 | 이슈 | 원인 | 수정 | 관련 파일 |
|------|------|------|------|----------|
| 2026-01-22 | 기존 이슈 | ... | ... | ... |
| 2026-01-22 | 새 이슈 | 새 원인 | 새 수정 | `new-file.ts` |
```

## 주의사항

- 이 스킬은 버그 수정 후, 커밋 전에 실행하세요
- 정보가 부족하면 사용자에게 질문합니다
